Codeforces-1088B

题目传送门

题意：每次删除最小的正数且让所有的数都减去这个数。进行m次操作，求每次删除数的大小。如果没有则输出0。

思路：输出的数一定是从小到大的，而相同的数减去一次后就都变成0了，所以排序去重一次后，每次删除的数就是a[i]-a[i-1]

Codeforces-1088C

题目传送门

题意：有两个操作,都是选定$i$和$x$，第一个是对$a_1,…,a_i$每个数加上$x$，第二个是每个数都$\mod x$。构造一种方法，进行不超过n+1次操作后序列是严格单调递增的。其中$a_i<=10^5$,$x<=10^6$。

思路：有两点需要注意：（1）如果$a[i]-x>0$, 那么$a[i] \mod (a[i]-x) = x$ （2）对于操作2 i x，如果$a_1,…，a_{i-1}$均小于x,那么相当于操作只对$a_i$有效。那么我们可以让$a_i=i$，方法是先1 i 800000，那么每个数都非常大。之后对每个$i$进行2 i a[i]-i。此时的a[i]-i一定大于i-1，因为之前每个数都加了800000，那么取模操作只对$a_i$有效，就可以对每个数分别单独进行操作了。正好用n+1次操作。

Codeforces-1088D

题目传送门
喜闻乐见我又双叒叕死在交互题上了，晚上深夜时刻还是脑子不好使$:($
题意：猜一个二元组$(a,b)$，其中$a,b < 2^{30}$。每次只能猜一个$(c,d)$，若$a \oplus c < b \oplus d$ 返回-1，相等返回0，大于返回1。要求猜不超过62次。

思路：数最多有30位二进制数，62正好是它的两倍，疯狂暗示逐位确定。但是大于和小于的关系不一定由当前位确定，所以要有一个合适的前缀去抵消前面二进制位的影响。E.g, 对于倒数第3位猜两次$(\overline{XXX}100, 000000)$和$(\overline{XXX}000, 000100)$，其中$\overline{XXX}$是为了抵消前面位数的影响。第三位上要异或的值是0, 1和1, 0。如果两次比较结果相同，意味着a,b在这一二进制位上数值不相同，否则是相同的。如果$a_i,b_i$是相同的，那么两次操作必然一个是1一个是-1，因为前面的二进制位被抵消了，而异或后这一二进制位（当前最高位）不相同。如果0,1的结果是1那么$a_i=1\ b_i=0$，否则$a_i=0\ b_i=1$。如果不相同那么找出上一次两个二进制位不相同的位置j（如果不存在这样的j就直接比较$a,b$），那么那一次第j位的两次询问都抵消了第j位的影响（都使得那一位均是0或1），根据那一次询问的值我们可以知道对于$suf_{j+1}$的大小关系，一定是-1或1，因为此时的“不相同的最高位”i是不相同的。显然如果是-1那么$a_i=0,b_i=1$，否则$a_i=1,b_i=0$，之后让前缀pre加上$2^i$，这个pre就是之前的$\overline{XXX}$，使得$X_i,…X_{29}$的影响被抵消。至此我们用两次比较确定了一个位置的$a_i$和$b_i$，加上最开始需要记录的大小关系总共猜61次。

Codeforces-1088F

题目传送门
一个很假的F题，感觉难度和D差不多
题意：给一棵树和每个点的权值$a_i$，让你构建一棵树使得整棵树的权值最小。权值的计算规则如下。$W = \sum_{i=1}^N{deg_i * a_i} + \sum_{for each (u,v) in new tree}min(a_u,a_v)·log_2{dist(u,v)}$其中$dist(u,v)$是原树中$u$到$v$的距离。边的距离都是1。给定的原树有一个重要的性质：对于任何一个不是最小点权的点，都存在一个相邻的点，且点权比他小。

思路：很像最小生成树的问题，只是考虑加点而不是加边。如果考虑围绕$min(a_u,a_v)$这个式子来计算权值，按点权从小到大加入树中，会发现新加入$a_u$到$a_v$上的权值是$min(a_v+a_v·log_2dist(u,v)) + a_u$ 那么每次加$u$进来只用考虑$v$与之的关系。在原树中考虑以最小的点权的点为根，会发现$a_u$的选择范围一定是自己的父亲到根节点上的路径。注意到由两点，（1）根节点是最小的点，如果选择向上到根节点再往外的点，那么和根节点相比距离更远点权还更大，一定是更亏的选择。（2）往下的点一定是点权更大的点，否则如果由两个点权更小的点相邻，那么根据给出的性质树就会有环。然后我们枚举$log_2dist(u,v)$，这个用树上倍增就可以解决了。总复杂度$O(nlogn)$。
